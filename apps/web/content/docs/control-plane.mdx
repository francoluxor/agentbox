---
title: Control plane
description: A hosted service that lets cloud boxes keep pushing and opening PRs with your laptop off
---

By default the host relay runs on your laptop and performs every privileged action a box can't do itself — `git push`, opening PRs, leasing credentials. Turn the laptop off and those stop for your cloud boxes.

The **control plane** is an optional hosted service that takes over the *centralized* concerns so cloud boxes don't depend on your laptop:

- **git credentials** — a GitHub App leases short-lived (1-hour), single-repo tokens to boxes on demand. The plane never holds a long-lived PAT and never your repo contents; a compromised box yields at most a 1-hour, single-repo token.
- **permission state** — pending approvals live in the plane's database; boxes poll for the verdict and you answer from anywhere (CLI or the deployment).
- **registry / events / status** — one cross-machine view of every box.

It is the same `@agentbox/relay` core wrapped as a **Next.js + Postgres app**. The identical code deploys to **Vercel** (managed) or **self-hosts on a VPS** (Postgres + `next start` via docker-compose) — only the host differs. It is stateless per request and does **no host execution**: host-local actions (`cp`, `download`, `checkpoint`) stay on the laptop relay; cloud boxes push to GitHub directly with a leased token.

<Callout type="info" title="Cloud boxes only">Only **cloud** boxes (hetzner, vercel, e2b, daytona) get the laptop-off guarantee. Local **docker** boxes still need the laptop relay for host-local actions (they bind-mount the host `.git`), so a docker box on a sleeping laptop is simply offline.</Callout>

## Set up the GitHub App

The plane leases push tokens through a GitHub App. Create one and write the deploy config in a single browser flow:

```bash
agentbox control-plane setup
```

This opens GitHub to create the App (Repository permissions: **Contents** + **Pull requests**, both Read & write), then writes `~/.agentbox/control-plane/control-plane.env` (the App id + base64 private key + a generated admin token, mode `0600`). Install the App on the repos your boxes work in when prompted.

## Deploy

The deploy env from `setup` feeds either host. Both create their database tables automatically on first request.

### Vercel

```bash
cd apps/control-plane
vercel link
vercel env add GITHUB_APP_ID
vercel env add GITHUB_APP_PRIVATE_KEY     # base64 or raw PEM
vercel env add AGENTBOX_RELAY_ADMIN_TOKEN
# attach a Postgres (e.g. the Neon integration) so POSTGRES_URL is injected
vercel deploy --prod
```

### Self-host (docker-compose)

```bash
cd apps/control-plane
docker compose --env-file ~/.agentbox/control-plane/control-plane.env up --build
# Postgres + the app on :8787
curl localhost:8787/healthz
```

## Point your boxes at it

```bash
agentbox control-plane set-url https://<your-plane-url>
agentbox control-plane status            # reachability check
```

`set-url` writes [`relay.controlPlaneUrl`](/docs/configuration). New cloud boxes then point at the plane for leasing, permission state, and the registry/events, and push to GitHub themselves with a leased token.

## Creating boxes from the plane

The plane can enqueue box creation (`POST /remote/boxes`, admin-bearer) and return a job id. A long-running **worker** drains that queue and provisions the boxes — it leases a token, clones the repo, and hands the checkout to the normal provider create path:

```bash
# Run alongside the plane (self-host) or as a scheduled job; needs the App key + provider creds
agentbox control-plane worker --store "$POSTGRES_URL"
agentbox control-plane worker --store "$POSTGRES_URL" --once   # drain once and exit
```

The worker provisions **cloud** boxes only.

## Security

- Leased tokens are per-repo, minimal permissions (contents + pull requests), 1-hour, and never persisted (in-memory cache only). The plane holds only the App private key.
- A GitHub installation token is repo-scoped, not branch-scoped, so the lease gate auto-approves only `agentbox/*` branches; any other branch needs approval. Branch protection on protected branches is the real backstop.
- The plane gates `/admin/*` and `/remote/*` on a constant-time admin bearer, fail-closed — it refuses to boot without one.

See [sync and git](/docs/sync-and-git) for the laptop-relay credential model and the cloud providers ([hetzner](/docs/hetzner), [vercel](/docs/vercel), [e2b](/docs/e2b), [daytona](/docs/daytona)).
